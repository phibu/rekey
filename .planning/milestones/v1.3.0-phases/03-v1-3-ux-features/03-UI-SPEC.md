## Spacing Scale

Declared values (all multiples of 4, mapped to MUI `theme.spacing` units where 1 unit = 8px):

| Token | Value | MUI Unit | Usage |
|-------|-------|----------|-------|
| xs | 4px | 0.5 | Icon gaps, inline padding, meter top-margin |
| sm | 8px | 1 | Compact element spacing, adornment gaps |
| md | 16px | 2 | Default field margin-bottom (`sx={{ mb: 2 }}`) |
| lg | 24px | 3 | Confirm-field margin-bottom (`sx={{ mb: 3 }}`) |
| xl | 32px | 4 | Card padding on small screens (`p: { xs: 3 }` → 24px; `sm: 4` → 32px) |
| 2xl | 48px | 6 | Vertical page padding (`py: 4` on outer Box = 32px; gap: 3 = 24px) |
| 3xl | 64px | 8 | Not used in existing code; reserved for future layout |

**Exceptions:**
- Logo image max-height: 48px (constrained via `maxHeight: 48` sx prop, not a spacing token)
- Favicon: 32×32px served as static file, not a spacing concern
- Policy panel internal rows: 8px vertical padding between policy lines (sm token)
- HIBP indicator strip below new-password field: 4px top-margin (xs token), same pattern as PasswordStrengthMeter (`mt: 0.5`)
- Clipboard countdown chip: 4px vertical, 8px horizontal padding (xs/sm tokens)

## Typography

Inherits from existing MUI theme (`fontFamily: '"Inter", "Roboto", "Helvetica Neue", Arial, sans-serif'`).
Source: App.tsx `buildTheme`.

**Focal point:** The card heading "Change Account Password" (`h5`, 24px, weight 600) is the primary visual anchor on the page. All other type scales descend from it.

| Role | Size | MUI Variant | Weight | Line Height | Usage |
|------|------|-------------|--------|-------------|-------|
| Body | 14px | `body2` | 400 | 1.5 | Helper text, policy panel lines, HIBP copy, branding usage text block, footer text, HIBP "powered by k-anonymity" note |
| Label | 16px | `body1` / TextField label | 400 | 1.5 | Form field labels, portal name in header |
| Heading | 20px | `h6` | 600 | 1.2 | App header (LockPersonIcon + product name) |
| Display | 24px | `h5` | 600 | 1.2 | Card heading ("Change Account Password") |

**Constraints:**
- Exactly 4 font sizes: 14, 16, 20, 24px. No `caption` variant — footer text and HIBP attribution use `body2` (14px) with `color="text.secondary"`.
- Exactly 2 weights: 400 (regular body/labels) and 600 (all headings and primary button). App header `h6` uses 600 (overriding MUI's default 700) to keep the scale to two weights.

## Color

Based on existing MUI theme palette. Source: App.tsx `buildTheme`.

| Role | Light mode | Dark mode | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `#f5f5f7` | MUI dark default (`#121212`) | Page background (`background.default`) |
| Secondary (30%) | `#ffffff` (MUI Paper) | `#1e1e1e` (MUI Paper dark) | Card surface, policy panel background, HIBP indicator background |
| Accent (10%) | `#0b6366` | `#0d7377` | Reserved-for list below |
| Destructive | MUI `error.main` (`#d32f2f`) | MUI `error.main` | Error alerts, HIBP breached indicator, form field errors |
| Warning | MUI `warning.main` (`#ed6c02`) | MUI `warning.main` | Approaching-lockout alert, clipboard countdown when < 5 s remain |
| Success | MUI `success.main` (`#2e7d32`) | MUI `success.main` | HIBP safe indicator, success alert, strength meter score 3–4 |

**Accent (`#0b6366`) reserved for:**
1. Submit button (`variant="contained"` → primary.main)
2. App header icon and text (LockPersonIcon + portal name)
3. Logo image tint — if operator provides no logo, the LockPersonIcon retains `color: primary.main`
4. Focused TextField outline (MUI default behavior on primary palette)
5. LinearProgress bar on strength meter when score ≥ 3 (uses `color="success"` — not accent; do not change)

**Operator branding color override (FEAT-001):**
- If `ClientSettings.brandPrimaryColor` is present and passes WCAG AA contrast check (≥ 4.5:1 on white / ≥ 3:1 on dark surface), inject it as `palette.primary.main`. Otherwise fall back to `#0b6366`.
- Validation happens at theme-build time in `buildTheme()`. Invalid hex values silently fall back to default.

## Component Inventory

New components introduced by this phase. All use MUI primitives only (no new npm packages).

### 1. BrandHeader (replaces hard-coded App.tsx header block)

Renders: operator logo (if configured) OR fallback LockPersonIcon + portal name.

| State | Render |
|-------|--------|
| No logo configured | `LockPersonIcon` (28px, `color: primary.main`) + Typography h6 portal name |
| Logo URL configured | `<img>` constrained to `maxHeight: 48px`, `maxWidth: 200px`, `objectFit: contain`; alt = portal name |
| Logo load error | Fall back silently to icon + text |

Logo `src` attribute must point to the upgrade-safe static path (e.g., `/assets/brand/logo.png` served from `wwwroot/assets/brand/`). Never inline base64 in `ClientSettings`.

### 2. AdPasswordPolicyPanel (FEAT-002)

Rendered above the new-password field when `ClientSettings.showAdPasswordPolicy === true`.

| State | Render |
|-------|--------|
| Loading | Single `<Skeleton variant="rounded" height={56} />` — same pattern as app skeleton |
| Loaded, policy available | MUI `Paper` with `variant="outlined"`, `sx={{ p: 1.5, mb: 2 }}`. Rows: min length, complexity flag, history count |
| AD query failed | Hidden entirely (fails closed) — no error message shown to user |

Layout within panel: MUI `List` (dense) with `ListItem` rows. Each row: `ListItemIcon` (CheckCircleOutline in `text.secondary`), `ListItemText` primary in body2.

Copy pattern for policy rows (configurable via `ClientSettings.adPolicyLabels`):
- Minimum length: "Minimum {n} characters"
- Complexity: "Must include uppercase, lowercase, number, and symbol"
- History: "Cannot reuse last {n} passwords"

### 3. HibpIndicator (FEAT-004)

Rendered below the new-password field (same slot as PasswordStrengthMeter, stacked below it).
Triggered on new-password field `onBlur`. Debounced at 400ms (longer than strength meter's 250ms to avoid duplicate computation bursts).

| State | Render |
|-------|--------|
| Idle (no blur yet, or field empty) | Nothing rendered |
| Checking | `<LinearProgress />` indeterminate, height 2px, below field; no text |
| Safe | `<Alert severity="success" icon={<ShieldOutlined />} sx={{ py: 0.5 }}>` — "No known breaches found." |
| Breached | `<Alert severity="error" icon={<GppBadOutlined />} sx={{ py: 0.5 }}>` — "Found in {count} data breach{count === 1 ? '' : 'es'}. Choose a different password." |
| Check unavailable (fail-open) | If `FailOpenOnPwnedCheckUnavailable=true`: nothing rendered. If `false`: `<Alert severity="warning">` — "Could not verify password safety. Proceed with caution." |

Security constraint: The SHA-1 prefix (first 5 hex chars) is sent to `GET /api/hibp/{prefix}`. Plaintext never leaves the client. Hash computed client-side using `crypto.subtle.digest('SHA-1', encoded)`.

`sx={{ mt: 0.5, mb: 1 }}` — 4px top, 8px bottom, consistent with PasswordStrengthMeter spacing.

### 4. ClipboardCountdown (FEAT-003)

Rendered as a MUI `Chip` below the generate-password button, only after `handleGenerate()` fires and `ClipboardClearSeconds > 0`.

| State | Render |
|-------|--------|
| Counting down (> 5 s remaining) | `<Chip size="small" icon={<ContentPasteOffOutlined />} label="Clipboard clears in {n}s" color="default" />` |
| Counting down (≤ 5 s remaining) | Same chip, `color="warning"` |
| Cleared | `<Chip size="small" icon={<CheckCircleOutline />} label="Clipboard cleared" color="success" />` shown for 2 s, then unmounted |
| User copied different content | Countdown silently cancelled, chip unmounted immediately |

Chip placed at `sx={{ mt: 1 }}` below generate-password icon button row inside the TextField adornment area.

## Interaction Contracts

### FEAT-001 — Operator Branding

| Element | Contract |
|---------|----------|
| Logo serving path | Static files under `wwwroot/assets/brand/` — upgrade-safe (not overwritten by publish) |
| Favicon path | `wwwroot/assets/brand/favicon.ico` — `<link rel="icon">` injected in `index.html` at build time |
| Portal name | Falls back to "PassReset" if `ClientSettings.portalName` is absent |
| Company name | Rendered as footer suffix: "© {year} {companyName}" replacing "Internal IT Tool" |
| Usage text | Rendered as `<Typography variant="body2" color="text.secondary" sx={{ mb: 2 }}>` above the form, replacing `form.helpText` if set |
| Primary color | Validated at `buildTheme()` time; invalid values silently fall back to `#0b6366` |

### FEAT-002 — AD Policy Panel

| Contract | Detail |
|---------|--------|
| Trigger | `ClientSettings.showAdPasswordPolicy === true` |
| Data source | `GET /api/password-policy` (new endpoint returning min length, complexity, history) |
| Failure mode | Panel hidden; no user-visible error; backend logs the AD query failure |
| Position | Immediately above new-password TextField, below current-password TextField |
| Accessibility | Panel `role="region"` with `aria-label="Password requirements"` |

### FEAT-003 — Clipboard Clearing

| Contract | Detail |
|---------|--------|
| Trigger | `handleGenerate()` fires AND `ClipboardClearSeconds > 0` |
| Implementation | `navigator.clipboard.writeText('')` only if `await navigator.clipboard.readText() === generatedPassword` |
| Fallback | If `clipboard.readText()` is denied by permissions policy, clear unconditionally after countdown |
| Countdown tick | `setInterval` at 1000ms, stored in ref, cleared on unmount |
| No clobber guarantee | Read-before-clear guard prevents clearing user content copied after generation |

### FEAT-004 — HIBP Blur Indicator

| Contract | Detail |
|---------|--------|
| Trigger | `onBlur` on new-password field, password non-empty |
| Debounce | 400ms from blur event |
| API | `GET /api/hibp/{sha1prefix}` — proxied server-side to `api.pwnedpasswords.com/range/{prefix}` |
| Plaintext guarantee | SHA-1 hash computed client-side via `crypto.subtle`; only 5-char prefix sent to server |
| Fail-open | If `FailOpenOnPwnedCheckUnavailable=true` and check fails, indicator hidden (no error shown) |
| Fail-closed | If `FailOpenOnPwnedCheckUnavailable=false` and check fails, warning Alert shown |
| Re-check | Re-triggers on every blur when value has changed since last check |

## Accessibility Contracts

| Element | Requirement |
|---------|-------------|
| Logo `<img>` | `alt` = portal name; if decorative, `alt=""` only if portal name appears adjacent in text |
| HibpIndicator Alert | `role="status"` on safe/cleared states; `role="alert"` on breached/warning states (MUI Alert uses `role="alert"` by default) |
| AdPasswordPolicyPanel | `role="region"` + `aria-label="Password requirements"` |
| ClipboardCountdown Chip | `aria-live="polite"` wrapper so screen readers announce countdown changes without interrupting |
| HIBP checking progress | `aria-label="Checking password breach status"` on LinearProgress |
| Color-only states | HIBP and clipboard indicators always pair color with icon + text (never color-only signaling) |
